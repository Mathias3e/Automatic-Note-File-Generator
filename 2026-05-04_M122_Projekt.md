## **M122 - Projekt - Mathias, Manuel, Leon**
---
#### **Projektbeschreibung**

Dieses Skript erstellt automatisch Markdown-Dateien auf Basis von vordefinierten Templates – speziell für Schüler, die jede Woche Unterricht in verschiedenen Modulen haben.
Für jedes Modul kann ein eigenes Template erstellt werden, z. B. mit Bereichen für Notizen, Fragen oder anstehende Termine. Das Skript erstellt diese Datei dann automatisch vor dem Unterricht – fertig benannt, fertig abgelegt, fertig strukturiert.
So muss man sich keine Gedanken mehr machen: Wie nenne ich die Datei? Wo speichere ich sie? Wo schreibe ich schnell etwas auf? Alles ist bereits vorbereitet und wartet auf Inhalte.

*Wir entwickeln vorerst für Windows, da verbreiteter. Wenn noch Zeit übrig ist, werden wir mithilfe von KI, ebenfalls eine Version für Linux bereitstellen.*

---
#### **Features / Funktionen**

**Templates**
- Beliebige `.md`-Dateien als Template definieren
- Platzhalter im Inhalt und im Dateinamen, die automatisch ersetzt werden (z. B. `{{date}}`, `{{modul}}`)
- Eigene Platzhalter mit festen Werten definierbar (z. B. `{{lehrer}}` → `Herr Müller`)
- Vordefinierte Bereiche im Template möglich: Notizen, Fragen, Termine, etc.

**Platzhalter**
- In der Template-datei können Platzhalter genutzt werden zwei geschweiften Klammern.
- z.B. {{date}}, {{modul}}
- Bei der Konfiguration kann dann gesagt werden durch was diese Platzhalter jeweils ersetzt werden sollen.
- {{date}}, {{day}}, {{week}} sind bereits vorhanden
	- {{date}} = fügt das Datum in dem Format ein, wie es in der Konfigurationsdatei angegeben wurde.
	- {{day}} = fügt den heutigen Tag ein (Format ebenfalls in der Konfigurationsdatei bestimmbar.)
	- {{week}} = fügt die Kalenderwoche (z.B. KW20) ein.

**Konfiguration**
- Mehrere Konfigurationen verwaltbar – eine pro Modul oder Fach
- Datumsformat frei wählbar (`YYYY-MM-DD`, `DD.MM.YYYY`, etc.)
- Ausgabeordner pro Konfiguration festlegbar
- Dateiname mit Platzhaltern definierbar (z. B. `2026-05-15_M122.md`)

**Automatisierung**
- Häufigkeit einstellbar: täglich, wöchentlich, monatlich – inkl. Uhrzeit
- Das Skript prüft ob für das aktuelle Intervall bereits eine Datei existiert → keine Duplikate
- Läuft im Hintergrund über den Windows Task Planner

**Bedienung über das Terminal**
- `skript -i` → Installiert alles Nötige
- `skript -k` → Öffnet die Konfigurationen zum Bearbeiten (bestehende Konfigurationen bearbeiten, neue Konfiguration erstellen)
- `skript` → Führt alle aktiven Konfigurationen aus (erstellt fällige Dateien)

---
#### **Geplanter Ablauf – Beispiel**

1. Skript starten mit `-i` (einmalige Installation)
2. Ordner festlegen, wo Dateien gespeichert werden dürfen (z.B. Für Konfigurationen (.json oder db (noch offen)))
3. Skript starten mit `-k` (Konfiguration)
4. Neue Konfiguration erstellen:
   - Template-Datei auswählen
   - Gewünschten Dateinamen definieren (z. B. `{{datum}}_M122.md`)
   - Ausgabeordner festlegen
   - Platzhalter definieren
   - Wiederholung festlegen (z. B. jeden Montag, 07:00 Uhr)
5. Ab sofort erstellt das Skript die Datei automatisch vor dem Unterricht

> **Hinweis:** Das Skript muss regelmässig ausgeführt werden, damit es prüfen kann ob eine neue Datei erstellt werden soll. Dies übernimmt der Windows Task Planner automatisch im Hintergrund.
